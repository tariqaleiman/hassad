## ✨ المميزات الرئيسية

### 🤝 الشراكة والملكية
- دعم أي عدد من الشركاء مع أي نسبة ملكية (50/50، 25/75، 0/100...)
- **3 مستويات منفصلة** لكل رأس: ملكية رأس المال — نصيب البيع — نصيب المواليد
- تقارير مالية مفصلة لكل شريك على حدة

### 🐄 إدارة القطيع
- تتبع كامل: حلوب ↔ جافة، حامل، عجول، تسمين
- تنبيه تلقائي لفترة الجفاف (60 يوم قبل الولادة)
- ملف تفصيلي لكل رأس: تاريخ الحالات، الصحة، التلقيح، المواليد

### 🍼 المواليد والملكية
- **تصحيح مشكلة الملكية:** ملكية العجل تُحسب تلقائياً من "نصيب المواليد" للأم
- عرض واضح قبل الحفظ مع إمكانية التعديل

### 🥛 الحليب
- تسجيل صباحي / مسائي / يوم كامل
- **تسجيل دفعة واحدة لكل الأبقار معاً، أو بقرة واحدة فقط** (تبديل سريع بين الوضعين)
- محاسبة تاجر الحليب مع توزيع الإيراد

### 🌿 المحاصيل والمراعي
- **تتبع مراحل الزراعة المتتابعة لكل محصول:** بذر/شتلات → نمو → تزهير → نضج → حصاد، مع تاريخ كل مرحلة
- برسيم وحشات (كل حشة منفصلة) مع تحديد المصير: علف / إيجار قيراط / بيع
- عمليات زراعية بوحدات مختلفة: ساعة، قيراط، شكارة، لتر، مرة رش، **يومية (أجر عامل) مع اسم العامل**
- مخلفات زراعية (بنجر، بطاطا) مع تكاليف النقل والتجهيز

### 💰 الديون والمالية
- **صفحة تفصيلية لكل مورد/دائن:** كل ديونه ودفعاته في ملف واحد
- تجميع الديون حسب المورد في صفحة الديون والتقارير المالية

### 🛡️ حماية البيانات
- نسخة احتياطية تلقائية (rolling backup) قبل كل حفظ — لو حصل خطأ في الحفظ، آخر نسخة سليمة محفوظة
- **تنزيل/استيراد نسخة احتياطية يدوية** بصيغة JSON من الشريط الجانبي
- ترقية تلقائية للبيانات القديمة عند كل تحديث للنظام — لا فقد لأي بيانات

### 📦 المخزون
- تتبع دقيق (وارد/صادر) لكل: أعلاف، أسمدة، مبيدات، وقود، قطع غيار
- تحويل بين أنواع الأعلاف (دراس: قش → تبن)
- تنبيه عند انخفاض المخزون عن الحد الأدنى

### 🔧 الصيانة
- تغيير زيت، صيانة جرار، بنزين، صيانة رشاش وري
- ربط بالديون تلقائياً للمشتريات الآجلة

## 🔥 قاعدة بيانات Firebase

النظام الآن متصل بـ Firebase Realtime Database كمصدر أساسي لحفظ البيانات، مع نسخة محلية فورية (localStorage) تعمل حتى بدون إنترنت وتتزامن تلقائياً عند العودة للاتصال. بياناتك القديمة المحفوظة محلياً تُرفع تلقائياً لأول مرة عند فتح النظام بعد هذا التحديث — لا فقد لأي بيانات.

## 🧾 المصروفات المستقلة عن العمليات

كل مصروف (مواد رش، بنزين، أجور عمالة، صيانة...) يُسجَّل ببند مستقل تماماً عن تاريخ تنفيذ العملية الزراعية، ويمكن ربطه اختيارياً بمحصول، عملية محددة، أو حيوان. هذا يسمح بتسجيل عملية رش بتاريخ، ومصروف المواد المستخدمة فيها بتاريخ آخر، وأكثر من بند تكلفة لعملية واحدة (مواد + بنزين + صيانة الماتور).

## 📊 النظام المحاسبي الكامل

شجرة حسابات حقيقية (أصول، التزامات، حقوق ملكية، إيرادات، مصروفات) تُبنى تلقائياً من كل عملية مسجلة في النظام — بدون أي إدخال يدوي إضافي. التقارير المتاحة من صفحة "التقارير المالية":
- **دفتر اليومية** — كل القيود بترتيب تاريخي
- **دفتر الأستاذ** — حركة كل حساب ورصيده الجاري
- **ميزان المراجعة** — تحقق من توازن الدفاتر
- **قائمة الدخل** — الإيرادات والمصروفات وصافي الدخل
- **الميزانية** — الأصول والالتزامات وحقوق الشركاء
- **تقرير الأعلاف مقابل اللبن** — تحميل تكلفة الأعلاف والصحة الأسبوعية لكامل القطيع على إيراد الحليب، لمعرفة الفائض أو العجز أسبوعياً

## 🌾 مرونة المحاصيل والعمليات

قائمة أنواع المحاصيل قابلة للتوسع بالكامل — اكتب أي نوع محصول جديد (مثل "ذرة شامية لب سوبر") وسيُحفظ تلقائياً للاستخدام لاحقاً. وحدة "يومية" متاحة في كل عمليات الزراعة لتسجيل أجور العمالة مع اسم العامل.



```
hassad/
├── index.html          # التطبيق الرئيسي
├── server.py           # سيرفر محلي احتياطي (Python) — Firebase هو المصدر الأساسي
├── assets/
│   └── logo.svg
├── css/
│   ├── all.min.css     # Font Awesome 6
│   ├── style.css       # التصميم الأساسي
│   └── app.css         # مكونات إضافية
└── js/
    ├── firebase.js     # الاتصال بـ Firebase (المصدر الأساسي للحفظ)
    ├── core.js         # البيانات والحفظ والأدوات المشتركة
    ├── partners.js     # الشركاء والملكية
    ├── cattle.js       # القطيع
    ├── milk.js         # الحليب
    ├── repro.js        # التلقيح والتناسل والولادات
    ├── calves.js       # العجول
    ├── health.js       # الصحة والرعاية
    ├── crops.js        # المحاصيل ومراحل الزراعة والعمليات والحشات
    ├── expenses.js     # المصروفات المستقلة (مرتبطة اختيارياً بمحصول/عملية/حيوان)
    ├── harvest.js       # الحصاد والمخلفات
    ├── inventory.js    # المخزون والصيانة
    ├── accounting.js   # شجرة الحسابات والدفاتر المحاسبية التلقائية
    ├── finance.js      # المالية والتقارير والديون
    └── app.js          # التوجيه والقائمة ولوحة التحكم
```

## 🔒 الخصوصية والبيانات

- **المصدر الأساسي:** Firebase Realtime Database (بياناتك محفوظة في حساب Firebase الخاص بالمشروع)
- **نسخة محلية فورية:** `localStorage` بمتصفحك — يعمل النظام حتى بدون إنترنت ويتزامن تلقائياً عند العودة للاتصال
- **نسخ احتياطية:** يمكنك تنزيل نسخة JSON كاملة من البيانات في أي وقت من الشريط الجانبي
